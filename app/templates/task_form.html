🕷️ Crawler dal seed (opzionale)

Se attivo: il runner parte dai seed URL, naviga i link interni del sito (BFS) e raccoglie tutti gli URL che corrispondono al pattern, poi li processa con LLM extraction. Il pattern URL è auto-rilevato dall'agente partendo dallo schema (1 chiamata LLM all'inizio): puoi sovrascriverlo manualmente con una regex Python se vuoi.

🧠 Quali LLM vengono usati e per cosa

Quando il crawler è abilitato il task fa 2 tipi di chiamate LLM molto diverse:

FaseQuandoDifficoltàChi la fa di default
1. Discovery (auto-detect URL pattern) 1 sola volta, all'inizio 🔥 Alta — ragionare sulla struttura del sito Il modello principale (sotto)
2. Extraction (estrazione campi da pagina) N volte (1 per URL discovered) 🟢 Bassa — riempire schema da testo Il modello principale (sotto)

💡 Per cataloghi grandi conviene usare un modello capace per Discovery (es. gpt-4o-mini, ~$0.0001 per 1 chiamata) e un modello locale gratis per Extraction (es. qwen3-coder:30b su Ollama). Risparmi 99% del costo. Compila i campi sotto SOLO se vuoi differenziarli; altrimenti lascia vuoti = il modello principale fa entrambe.
