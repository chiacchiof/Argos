➕ Nuovo asset manuale

Aggiungi un asset al DB senza passare da uno scraping. Utile per inserire manualmente record di ricerca, lead one-off, o entità a cui collegare contatti.

{# === Campi rapidi: il modo consigliato per inserire whatsapp/email/role/ecc. senza scrivere JSON a mano. Le chiavi note (whatsapp/email/telegram/...) vengono promosse a colonne dell'asset dal backend; le altre diventano tag. === #}
📋 Campi rapidi (consigliato — niente JSON da scrivere)

Aggiungi righe key=value. Le chiavi standard (whatsapp, email, telegram, display_name, sitoweb, role, organization, ...) sono in elenco. Puoi anche scrivere chiavi custom nel campo (lowercase, a-z 0-9 _ -). Salvate come colonne dedicate dell'asset (le standard) o come tag attributi (le altre).

🏷️ Tag attributi avanzati (opzionale, fino a 5 — vincolato a key lowercase a-z 0-9 _ -)

Per tag arbitrari come municipality:Aci Bonaccorsi, category:palestra. Equivalente al blocco "Campi rapidi" sopra ma con vincolo stretto sulle key. Tag aggiuntivi modificabili dopo in Edit.

🔍 Raw JSON (opzionale, solo se hai bisogno di payload nidificato / array)

Per casi avanzati: payload nidificato, array, dati custom non rappresentabili come coppie key=value. Per la maggior parte dei casi usa "Campi rapidi" sopra (piu' semplice, niente sintassi JSON da rispettare).

Formato: dict JSON valido con virgolette dritte (", non “ ”). Esempio: 

{
  "whatsapp": "+393294257497",
  "email": "francesco@edg.com",
  "organization": "EDG",
  "role": "CEO",
  "social": [
    {"platform": "instagram", "url": "https://instagram.com/francesco"}
  ],
  "altri_dati": [1, 2, 3]
}

Le chiavi note (whatsapp, email, telegram, display_name, sitoweb, social) vengono promosse a colonne dell'asset. Le altre diventano tag se il nome rispetta ^[a-z][a-z0-9_-]{0,49}$.
